# TypeScript reference — bond op golden vectors

This file pins the canonical wire bytes for every `BondOp` variant so the Rust and TypeScript implementations stay byte-for-byte identical. Both reference vectors are deterministic — same seed inputs, same bytes, every time — and asserted in `mfn-consensus` unit tests. Any drift here is a consensus-breaking change.

The values are generated by **`cloonan-group`** `scripts/smoke-bond.ts` (`GOLDEN_BOND_OP_*` and `GOLDEN_UNBOND_OP_*`) and mirrored by the Rust tests below.

## `BondOp::Register` — burn-on-bond registration

**Inputs (deterministic):**

| Input | Value |
|-------|-------|
| `stake` | `1_000_000` |
| `vrf_pk` | `7 · G` (Ed25519 generator multiplied by scalar 7) |
| `bls_pk` | BLS keypair from 48-byte seed `[1, 2, 3, …, 48]`, public key only |
| `payout` | `None` |

**Reference bytes:**

| Field | Hex |
|-------|-----|
| Canonical `encode_bond_op` wire | `0000000000000f4240b862409fb5c4c4123df2abf7462b88f041ad36dd6864ce872fd5472be363c5b1aab6e7afc31b3d67eef05ff38bfb40d5e608f352b3c0341ec019653505d7c1f13dd1e60640bb00d0735daa5cbd3b902600` |
| `bond_op_leaf_hash` (32 bytes) | `51164109143ca1e9db57a1738443c078389c6492e5ea14ed8ecf0aea83d1962b` |

Rust assertion: `bond_wire::tests::bond_register_wire_matches_cloonan_ts_smoke_reference` in `mfn-consensus`.

## `BondOp::Unbond` — BLS-signed exit authorization

BLS signatures over BLS12-381 are `sig = sk · H(m)`. They are **deterministic** (no nonce), so the wire bytes are fully reproducible from the keypair seed and `validator_index` alone — no RNG involvement, no environmental state.

**Inputs (deterministic):**

| Input | Value |
|-------|-------|
| `bls_keypair` | `bls_keygen_from_seed([1, 2, 3, …, 48])` — *same seed as `BondOp::Register`* |
| `validator_index` | `7` (`u32`, big-endian on the wire) |
| Signed payload | `dhash(MFBN-1/unbond-op-sig, validator_index.to_be_bytes())` |
| `sig` | `sign_unbond(validator_index, &bls.sk)` |

**Reference bytes:**

| Field | Hex |
|-------|-----|
| Canonical `encode_bond_op` wire | `0100000007a23607ffd488bbf50edaa45790118204321064a895ba974faf132337b21cc0190d1bf3e7d82399b7b954f310860aa9fd06cf898bce3b2a7731f685345b52f2008ecb526ec415694e665599d8859d3068bef8a4a0a98ff0a1e873acf82fd4e1e0` |
| `bond_op_leaf_hash` (32 bytes) | `6a51ca2c8e53443cfa9cdb096096097bcba3428098b628778cf55602207a5833` |

Wire structure (101 bytes total): `tag(1) ‖ validator_index(4, BE) ‖ bls_signature(96)`. Tag byte `0x01` = `BOND_OP_UNBOND`.

Rust assertion: `bond_wire::tests::bond_unbond_wire_matches_cloonan_ts_smoke_reference` in `mfn-consensus`. The test also re-verifies `sig` against the reference public key, guarding against any future change to either the BLS hash-to-curve domain or the unbond signing-payload construction.

## Notes

- **Genesis `bonding_params`** are optional on `GenesisConfig` (`Option<BondingParams>`). `None` falls back to `DEFAULT_BONDING_PARAMS`, matching the TS `ChainStore` / `applyGenesis` behavior.
- The same 48-byte seed (`1..=48`) is reused across both vectors so a single TS smoke fixture covers both `BondOp` arms.
- Domain separation: the unbond signing payload is `dhash(MFBN-1/unbond-op-sig, …)`, distinct from `MFBN-1/bond-op-leaf` (Merkle leaf domain) and from every other consensus tag — a leaked unbond signature cannot be replayed for any other purpose, including a second unbond at a different `validator_index` or on a forked chain.
