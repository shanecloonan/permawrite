# TypeScript reference — bond op golden vectors

This file pins the canonical wire bytes for every `BondOp` variant so the Rust and TypeScript implementations stay byte-for-byte identical. Both reference vectors are deterministic — same seed inputs, same bytes, every time — and asserted in `mfn-consensus` unit tests. Any drift here is a consensus-breaking change.

The values are generated by **`cloonan-group`** `scripts/smoke-bond.ts` (`GOLDEN_BOND_OP_*` and `GOLDEN_UNBOND_OP_*`) and mirrored by the Rust tests below.

## `BondOp::Register` — burn-on-bond registration (M1.5 authenticated wire)

As of M1.5, `BondOp::Register` is BLS-authenticated by the operator's own voting key. The wire grew a 96-byte signature trailer and the pre-M1.5 vector is **retired** — any TS implementation must regenerate against the new authenticated format. Both arms of `BondOp` now share the same `bls_keygen_from_seed([1..=48])` convention, so a single TS smoke fixture covers both.

**Inputs (deterministic):**

| Input | Value |
|-------|-------|
| `bls_keypair` | `bls_keygen_from_seed([1, 2, 3, …, 48])` — *same seed as `BondOp::Unbond`* |
| `stake` | `1_000_000` (`u64`, big-endian on the wire) |
| `vrf_pk` | `7 · G` (Ed25519 generator multiplied by scalar 7) |
| `payout` | `None` |
| Signed payload | `dhash(MFBN-1/register-op-sig, stake ‖ vrf_pk ‖ bls_pk ‖ payout_flag ‖ [payout?])` |
| `sig` | `sign_register(stake, &vrf_pk, &bls.pk, None, &bls.sk)` |

**Reference bytes:**

| Field | Hex |

| Canonical `encode_bond_op` wire | `0000000000000f4240b862409fb5c4c4123df2abf7462b88f041ad36dd6864ce872fd5472be363c5b191cea2c39bbe275cc495b90b926c1e621df9d07624282c1ba157a12e97de284fb6327dc7a1165119d344721b382144ff00877cdf932aa770293b32e3412ba49c514f022108743153e9297d92d9fe3c9d08972c9fe41154b084d6c13c67e461add4015464f9be27c100f603555984c659d6c38d00e2cae23ae1c8d9f73a5cd23cd6297965fce9dbe9393e5dfb9e6b40d7e6` |
| `bond_op_leaf_hash` (32 bytes) | `01ff3ac647d6cfbab3e4d242838f472e3bcd818364865246d67a83c8c317af15` |

Wire structure (186 bytes total): `tag(1) ‖ stake(8, BE) ‖ vrf_pk(32) ‖ bls_pk(48) ‖ payout_flag(1) ‖ [view_pub(32) ‖ spend_pub(32)]? ‖ sig(96)`. Tag byte `0x00` = `BOND_OP_REGISTER`.

Rust assertion: `bond_wire::tests::bond_register_wire_matches_cloonan_ts_smoke_reference` in `mfn-consensus`. The test also re-verifies `sig` against the operator's public key, guarding against any future drift in either the BLS hash-to-curve domain or the register signing-payload construction.

## `BondOp::Unbond` — BLS-signed exit authorization

BLS signatures over BLS12-381 are `sig = sk · H(m)`. They are **deterministic** (no nonce), so the wire bytes are fully reproducible from the keypair seed and `validator_index` alone — no RNG involvement, no environmental state.

**Inputs (deterministic):**

| Input | Value |
|-------|-------|
| `bls_keypair` | `bls_keygen_from_seed([1, 2, 3, …, 48])` — *same seed as `BondOp::Register`* |
| `validator_index` | `7` (`u32`, big-endian on the wire) |
| Signed payload | `dhash(MFBN-1/unbond-op-sig, validator_index.to_be_bytes())` |
| `sig` | `sign_unbond(validator_index, &bls.sk)` |

**Reference bytes:**

| Field | Hex |
|-------|-----|
| Canonical `encode_bond_op` wire | `0100000007a23607ffd488bbf50edaa45790118204321064a895ba974faf132337b21cc0190d1bf3e7d82399b7b954f310860aa9fd06cf898bce3b2a7731f685345b52f2008ecb526ec415694e665599d8859d3068bef8a4a0a98ff0a1e873acf82fd4e1e0` |
| `bond_op_leaf_hash` (32 bytes) | `6a51ca2c8e53443cfa9cdb096096097bcba3428098b628778cf55602207a5833` |

Wire structure (101 bytes total): `tag(1) ‖ validator_index(4, BE) ‖ bls_signature(96)`. Tag byte `0x01` = `BOND_OP_UNBOND`.

Rust assertion: `bond_wire::tests::bond_unbond_wire_matches_cloonan_ts_smoke_reference` in `mfn-consensus`. The test also re-verifies `sig` against the reference public key, guarding against any future change to either the BLS hash-to-curve domain or the unbond signing-payload construction.

## Notes

- **Genesis `bonding_params`** are optional on `GenesisConfig` (`Option<BondingParams>`). `None` falls back to `DEFAULT_BONDING_PARAMS`, matching the TS `ChainStore` / `applyGenesis` behavior.
- The same 48-byte seed (`1..=48`) is reused across both vectors so a single TS smoke fixture covers both `BondOp` arms.
- **Domain separation.** Every BLS-signed authorization payload carries its own consensus tag so a leaked signature cannot be replayed across roles:
  - `dhash(MFBN-1/register-op-sig, …)` for `BondOp::Register` — binds `(stake, vrf_pk, bls_pk, payout)` together, defeating replay against a stranger's keys.
  - `dhash(MFBN-1/unbond-op-sig, …)` for `BondOp::Unbond` — binds the `validator_index` only; replay across the same validator is independently blocked by `pending_unbonds` rejecting duplicate enqueues.
  - Both are distinct from `MFBN-1/bond-op-leaf` (the Merkle leaf domain) and from every other consensus tag.
- **M1.5 wire-format break.** The pre-M1.5 unauthenticated `BondOp::Register` vector (no `sig` trailer) is retired. Any TS implementation that pinned the old bytes must regenerate against the M1.5 layout. There is no live chain to migrate.
